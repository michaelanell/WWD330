  • Chapter 2: Programming Basics


    • Primitive data types:
      • String
      • Symbol
      • Number - just number, not float or int
      • Boolean
      • Undefined
      • Null
    • Declaring Variables:
      • Const - for variables ment to stay constant
      • Let - for variables allowed to change
      • Let and const have block scope
    • Naming Conventions in JavaScript
      • Can start with upper or lower case letters, an underscore, dollar sign
      • Cannot contain numbers at start, but can contain them in other parts of name
      • Names that start with an underscore usually refer to private properties and methods
      • Variable names area case sensitive
      • Camel case is propobably best to use for naming, but underscore separated words are also often used

  • Chapter 2 - Arrays


    • Arrays:
      • To declare an array in JavaScript:
        • const arrayName = []; - dont have to declare size
        • Initializing a singe element
          • arrayName[0] = 'value1';
        • Array Literals
          • literal - notation for representig a fixed value in source code
          • arrayName = ['Value 1', 'Value 2', 'Value 3', 'Value 4', 'Value 5','Value 6'];
        • You can add any type of value to an array, one array can contain a variety of data types:
          • arrayName = ['Value 1', 2, [], true];
        • Array Operations:
          • Removing values:
            • The delete operator removes array items:
              • delete arrayName[3] //returns undefined
              • The value at place arrayName[3] will be removed from the array and be replaced with value undefined - The space myArray[3] will still exist in the array, just the value is removed and replaced
            • .length method - returns length of method
            • .pop() method - removes the last item from an array
            • .shift() method - removes the first item in an array
            • .push method - appends new value to end of array
            • .unshift() method - appends new items to beginning of array
            • .conacat method - merge arrays
            • .join() method - turn an array into a string that contains all the items in the array separated by commas.
            • .slice() method - creates a sub array from a 'slice' of the orginal array. It takes two parameters - the first number refers to the item posistion of the array that you would like to start the slice from. The second refers to the item position of the array that you would like to end the slice on.
            • .splice() method - removes items from an array then inserts new items into their place. It takes 3 parameters. The first two are numbers that dictate the to and from item positions in the array that you would like to remove. The third is the item value that is to replace the ones you remove. You can add more than one parameter for this purpose, and each item will be put in place of the ones you removed in order.
            • .reverse() method - reverses order of array
            • .sort() method - sorts the order of array
            • .indexOf() method - checks if a value exists in an array. If the value is there the (first occurance of the) index of the item will be returned.
          • Mutidimentional Arrays - an array of arrays

  • Designing Intuitive and Mobile Friendly Forms


    • Make things simple so users can understand them easily.
    • Most web forms are too long. Something to do to make froms shorter is to use hints instead of labels. Using float labels also helps users.
    • Make fields bigger for users to easily touch.
    • Entering information to a mobile device is a tedious and error prone process that leads users to abandon the form. Enabling autofill makes the forms quicker to fill out, reduces errors, and makes users more likely to complete the form.
    • Passwords matter - Entering passwords are especially difficult on mobile devices. Masking passwords is something that can be done away with on mobile to decrease user frustraion. Give the user the option to mask the passord or not with the eye symbol in the password field.
    • Numbers matter - auto advancing credit card numbers, and formatting spaces between number groups helps users input their infomation.
      When taking numbers ensure the number keypad is used on mobile. Default keyboard make inputting numbers tedious.
    • Calls matter - Dont force a copy and paste in order for users to call your number.

  • Designing a Better Mobile Advertising User Experience


    • Pop-up advirtisments are the most detested advertisements to users. When ads get in the way of content you loose engagment. If you are going to place ads in your site, make them small, dont let them take up 40-50% of your screen space. Video ads have higher converstion rates than pictures.
    • Things to remember:
    • Stop using pop-ups and pop-overs
    • Users dont visit your website to look at adds, they came for content.
    • If you use ads, make them unobstrusive. Your content comes first.
