Client-side Form Validation

  • Ensuring that user forms are filled out in the correct format.
  • Important to let a user know straigt away if they need to fix an input.
  • The main reasons form validation is important:
    • What to collect the right data in the correct format.
    • What to protect user's data - eg. having many password rules to make passwords secure.
    • Protect applications from malicious users who want to misuse unprotected forms.
  • Types of client side validation:
    • Built in form validation - uses HTML 5, doesn't require much JS. Has better performance than js. Not as customizable though.
      • Built-in form validation is created using validation attributes on HTML elements. These attributes include:
        • required - makes a field required
        • minlength - specifies min length of textual data input
        • maxlength - specifies max length of textual data input
        • min - specifies min values of numerical input
        • max - specifies max values of numerical input
        • type - specifies what type of input the data needs to be. eg. email address, number
        • pattern - specifies a regular expression that defines a pattern the input needs to follow
          • Regular expression (regex) - "is a pattern that can be used to match character combinations in text strings"
        When a form is completed and matches all these rules, the input is considered valid.
      • When an element is valid:
        • "The element matches the :valid CSS pseudo-class, which lets you apply a specific style to valid elements."
        • "If the user tries to send the data, the browser will submit the form, provided there is nothing else stopping it from doing so (e.g., JavaScript)."
      • When an element is invalid:
        • "The element matches the :invalid CSS pseudo-class, and sometimes other UI pseudo-classes (e.g., :out-of-range) depending on the error, which lets you apply a specific style to invalid elements."
        • "If the user tries to send the data, the browser will block the form and display an error message. "

    • Why you would want to use JS for you form validation:
      • You want to customize the look and feel of you form's error messages
      • Address the issue of legacy browsers that do not support HTML's built-in form validation
