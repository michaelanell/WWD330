Errors, Exceptions, and Warnings


  • Errors


    When there is an error in your program, it is usually caused by one of three things:
    • System error - issue with system/external device the program is being played on.
    • Program error - syntax or logic errors.
    • User error - user entered data that the program is unable to handle.

  • Exceptions


    • An error that produces a return value that can help deal with the error.

  • Stack Traces


    • Sequence of functions/method calls that find the point where the error occured in the program.
    • Works backwards from where the error occured to find the function or method that started the error sequence.

  • Warnings


    • Warning occurs when there is an error that isn't enough to cause program to crash.
    • Although the warning doesn't crash the program, it is good to resolve the warning errors because they can result in the program running incorrectly if you do not.

  • Strict Mode


    • Came out with ECMAScript 5.
    • Strict mode is a way of writing js that produces more exceptions and warnings and prohibits using depreciated features. Using strict mode results in code that is not sloppy, and is rather robust, clear, and fast code.
    • To write in strict mode add the following line of code to the start of a js file:
      • 'use strict';
    • If the js engine you are using supports strict mode, it will pick up this line of code. If not, it will be ignored.
    • You can apply strict mode to individual functions in your program by doing the following:
      • function strictly(){
        'use strict';
    • It is recommened that you invoke strict mode by placing all of your code in a self-invoking function that is set to use strict mode:
      • (function() { 'use strict'; // All your code would go inside this function }());

  • Exception Handlers - try, catch, finally


  • If you are unsure if a line of code will work and want to test it, put the code inside of a try{} block.
  • Create a catch{} block to that will run if the code inside of the try{} block throws an exception. To create a catch block, pass the error to the catch block and write what you want to happen if an excception is thrown by the try block inside of the catch block.
  • Lastly, to end your exception handler, add a final block after your catch block, that runs whether or not an exception occurrs.
  • Example:
    function imaginarySquareRoot(number) { 'use strict'; let answer; try { answer = String(squareRoot(number)); } catch(error) { answer = squareRoot(-number)+"i"; } finally { return `+ or - ${answer}`; } }
