Week One Notes

Master Mobile UX Sitepoint

  • Lesson 1 - Rewrite Brain for Mobile First


    • Start with designing for the smallest device that your code will run on instead of desktop.
    • "Whatever your are doing do mobile first" Eric Schmidt. Mobile is the most used device to view content on, far surpases laptop and desktop.
    • "Just because your website is responsive, it doesn't mean that it is mobile first."
    • Responsive Retrofitting - taking what you have already created for desktop and making it fit on mobile. This does address some important issues such as:
      • Touch interaction issues
      • Screen real estate
    • There are easy and difficult things to designing mobile first
    • Downsides:
      • Time consuming
      • Costly if you are rebuiling a site that already exists for desktop
    • Upsides
      • Taking time to redesign a site for mobile gives your users a much better experiance across different devices.
      • Address size constraints of mobile size and bandwidth.
      • Makes application better ready for future growth.
    • To Enforce Importance of Mobile First:
      • Desktop sales declining since 2011
      • Consumers purchase smaller devices of varying sizes at a high rate with a high turnover
      • Companies that manufacture devices make few desktops and many more smaller devices such as laptops, tablets, and smartphones.
      • Mobile is not a fad.
